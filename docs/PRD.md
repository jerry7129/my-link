# 프로젝트명: 마이링크 (My Link)

## 1. 프로젝트 개요
- **목적**: 여러 개의 분산된 소셜 미디어, 포트폴리오, 웹사이트 링크를 단 하나의 URL로 통합하여 개성 있게 공유할 수 있는 링크 모음 서비스
- **대상 사용자**: 
  - 다양한 플랫폼을 운영하는 크리에이터 및 인플루언서
  - 포트폴리오와 이력서를 한 곳에 모아두고 싶은 취업준비생
  - 여러 비즈니스 채널을 고객에게 한 번에 안내해야 하는 소상공인

---

## 2. 주요 기술 스택 구축 방안
- **프론트엔드**: Next.js, Tailwind CSS (shadcn/ui 기반 컴포넌트 라이브러리 사용)
- **비용 정책**: 100% 무료 서비스
- **백엔드/데이터베이스**: Firebase 생태계
  - **인증(Auth)**: Firebase Authentication (Google 소셜 로그인 전용)
  - **데이터베이스**: Firebase Firestore
  - *(이미지 업로드 기능이 제거됨에 따라 Firebase Storage는 사용하지 않음)*

---

## 3. 핵심 기능 목록 및 상세 설명

### 3-1. 회원가입 및 프로필 접속 주소 할당
1. **Google 소셜 로그인**
   - 최초 로그인 시 주소창에 쓰일 **단일 식별자(displayName)** 중복 확인 및 등재 필수 진행
     - **초기값 자동 부여**: 로그인한 구글 지메일 아이디의 `@` 앞부분을 초기값으로 제안
   - 핸들(`@`)이나 별도의 식별 기호 없이, 설정한 **displayName 자체가 고유 접속 URL**이 됨 (예: `jerry7129`인 경우 `mylink.com/jerry7129` 형태의 주소 채택)

### 3-2. 프로필 관리 시스템
1. **인라인 편집(Inline Editing) 지원**
   - 별도의 '수정 페이지'나 폼 전환 없이, 관리 화면에 보이는 텍스트를 클릭하여 제자리에서 바로 고치고 반영하는 인라인 편집 UX 제공
2. **이름(username) 및 프로필 소개 지원**
   - 사용자 식별을 위해 프로필 상단에 노출되는 **실제 이름(username)** 등록 지원
     - **초기값 자동 부여**: 로그인한 구글 계정의 프로필 이름(Profile Name)을 초기값으로 자동 세팅
   - 프로필 소개글(Bio) 텍스트 입력 지원
3. **사용자 프로필 이미지 (Google 프로필 연동)**
   - 별도의 사용자 프로필 이미지 직접 업로드 기능은 제공하지 않음
   - Google 소셜 로그인 시 제공받는 프로필 이미지(photoURL)를 프로필 화면에 기본으로 노출

### 3-3. 링크 관리 시스템 (CRUD)
1. **링크 아이템 기본 생성**
   - 연결 목적지 URL과 화면 버튼에 노출될 텍스트 제목만 단출하게 입력
2. **자동 아이콘 적용 (Google Favicon API)**
   - 입력한 대상 URL을 기반으로 **Google Favicon API**를 호출하여 해당 웹사이트의 파비콘 이미지를 렌더링, 링크 버튼 좌측 아이콘으로 자동 표시
3. **링크 수정 및 삭제**
   - 작성된 링크의 제목이나 변경된 URL을 수정하거나, 필요 없는 링크 아이템을 삭제
   - *(링크 드래그 앤 드롭 노출 순서 변경, 숨김/비활성화 기능은 지원하지 않음)*

### 3-4. 사용자 랜딩 페이지 (퍼블릭 방문 뷰)
1. **모바일 최적화 프로필 뷰**
   - 외부에서 링크를 클릭해 접속했을 때 조회되는 방문자 전용 화면
   - 화면 상단: 구글 프로필 이미지(photoURL) 및 실제 사용자의 이름(username)과 한 줄 소개(Bio) 노출
   - UI: shadcn/ui 기반의 단일적이고 깔끔한 텍스트 위주의 테마 제공
2. **링크 리스트 노출**
   - 데이터베이스에서 불러온 기본 링크 블록(구글 파비콘 아이콘 + 제목)들을 세로형 리스트로 나열

## 4. 데이터베이스 모델링 (Firebase Firestore)

Firestore를 기반으로 사용자의 정보와 링크 목록을 아래와 같은 계층형 구조로 저장합니다. 

### `users` 컬렉션 (사용자 정보)
각 사용자의 프로필 기본 정보를 담는 최상위 컬렉션입니다.
- **문서 ID (Document ID)**: `uid` (Firebase Auth 사용자 고유 식별자)
- **필드 (Fields)**:
  - `displayName` (String): 중복이 허용되지 않는 고유 식별자, 프로필 URL 경로로 적용됨 (초기값: 구글 이메일의 `@` 앞부분)
  - `username` (String): 화면에 보여질 실제 사용자의 이름 (초기값: 구글 프로필 이름)
  - `photoURL` (String): 구글 로그인에서 가져온 프로필 이미지 URL
  - `bio` (String): 프로필 하단에 표시될 한 줄 소개 글
  - `email` (String): 사용자 구글 계정 이메일
  - `createdAt` (Timestamp): 가입 일시

### `links` 서브 컬렉션 (해당 사용자의 링크 목록)
해당 사용자가 등록한 링크 아이템들을 저장하는 하위 컬렉션입니다.
- **경로**: `users/{uid}/links`
- **문서 ID (Document ID)**: 자동 생성되는 무작위 ID (`linkId`)
- **필드 (Fields)**:
  - `title` (String): 링크 클릭 버튼에 노출될 텍스트 제목
  - `url` (String): 바로가기 목적지 URL (Google Favicon API 연동 기준점)
  - `createdAt` (Timestamp): 링크 생성 일시 (기본 정렬 순서로 활용)
